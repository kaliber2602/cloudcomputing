Hệ thống mô phỏng Cloud Platform gồm đúng 9 loại máy chủ chạy trong container riêng biệt, kết nối qua mạng cloud-net.🖥️ Server 1: Web Frontend Server (Nginx)Vai trò: Static Website Hosting.Cấu trúc thư mục: /usr/share/nginx/html/.Nâng cao: * Thư mục /blog/ chứa ít nhất 3 bài viết: blog1.html, blog2.html, blog3.html.index.html trang chủ phải liệt kê danh sách bài viết bằng hyperlink.Tích hợp: Chứa thêm file database.html (giao diện hiển thị dữ liệu từ MariaDB).⚙️ Server 2: Application Backend Server (Python Flask/FastAPI)Vai trò: Xử lý logic và cung cấp REST API.Cổng nội bộ: 8081.Nâng cao (Tập trung logic):API /hello: Trả JSON chào mừng.API /student: Trả danh sách 5 sinh viên từ file students.json.API /students-db: Kết nối tới relational-database-server để thực hiện truy vấn SELECT dữ liệu thực tế.API /secure: Kiểm tra Access Token (OIDC) từ Keycloak để bảo mật tài nguyên.🗄️ Server 3: Relational Database Server (MariaDB)Vai trò: Lưu trữ dữ liệu quan hệ (RDBMS).Cấu hình:Mật khẩu Root: root.Database chính: studentdb.Volume: Ánh xạ thư mục ./init vào /docker-entrypoint-initdb.d để tự động chạy file .sql khởi tạo khi container bật lần đầu.Nâng cao: Bảng students phải có cấu trúc: id, student_id, fullname, dob, major và sẵn ít nhất 3 bản ghi.🔑 Server 4: Authentication & Identity Server (Keycloak)Vai trò: Identity Provider (IdP) quản lý Single Sign-On (SSO).Cấu hình Docker: Chạy image keycloak, lệnh start-dev, biến môi trường KC_BOOTSTRAP_ADMIN_USERNAME/PASSWORD.Nâng cao: * Tạo Realm mới theo mã số sinh viên (ví dụ: realm_sv01).Tạo 2 người dùng (sv01, sv02) và 1 Client (flask-app).Cung cấp URL Token Endpoint cho App Server thực hiện xác thực /secure.
